# ログ記事執筆スキル

## 既存記事の重複チェック

新しい記事を書く前に、`src/content/log/` 内のファイルを確認し、テーマや内容が既存記事と重複しないかを確認すること。

## ファイル作成

`src/content/log/{slug}.mdx` を作成する。フロントマター形式は README.md の「コンテンツの追加」を参照。

- **slug**: 記事の内容を英語で表したケバブケース（例: `clinic-google-maps-embed`）
- **date**: 作成日（今日の日付）を `YYYY-MM-DD` 形式で記入

## 文体・スタイル

- 対象読者: クリニック・医療機関の院長（医療以外のテーマの場合はそのテーマの読者に合わせる）
- 文体: 丁寧語（です・ます調）、句読点あり
- トーン: 押しつけがましくない。事実と論点を整理して提示し、読者が自分で判断できる書き方
- 専門用語を使う場合は簡単に説明を添える
- 数字・具体例を使って説得力を持たせる。データを使う場合は必ず WebSearch で出所を確認してから使用すること
- 締めの段落はFlagshipWorksへの相談を促す（医療テーマは「現役医師とWebデザインの専門家の視点で〜」、それ以外はテーマに合わせた文言にする）
- タグは README.md の「LogCta のタグ判定ルール」を参照して設定する

## データ・引用元の扱い

- 出所が確認できるデータのみ使用する
- 記事末尾に `## 参考` セクションを設け、使用したデータのURLをMarkdown記法のリンクで記載する（外部リンクは `MdxContent.tsx` が自動で新タブ開きにするため、HTMLタグは不要）
- 調査年が古い場合（3年以上）は年を明示する（例: 「2021年調査」）

## 記事構成の目安

- `##` 見出しを3〜5個
- 各セクション 150〜300字程度
- 必要に応じてリストやテーブルを使う
- 全体 800〜1,500字が目安

## 記事作成後にやること

1. `npm run build` でビルドエラーがないか確認
2. Notion の「ログ記事管理」データベースに追加（ステータス: 公開済み、slug・公開日・タグを入力）
3. ユーザーに内容確認を促す。push は明示的に指示があってから行う
